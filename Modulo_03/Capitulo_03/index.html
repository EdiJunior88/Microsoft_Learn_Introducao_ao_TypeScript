Exercício – declarar uma interface e criar uma instância dela no TypeScript

Para declarar uma interface, comece com a palavra-chave interface seguida pelo nome da interface (o identificador). O nome da interface não pode ser um dos nomes de tipo pré-definidos no sistema de tipos. E, por convenção, os nomes de interface são escritos em PascalCase.

Observação

As diretrizes de codificação do TypeScript sugerem que as interfaces não devem começar com a letra I.

Em seguida, defina as propriedades (ou membros) da interface e os respectivos tipos. As propriedades podem ser obrigatórias, opcionais ou somente leitura.

Tipos de propriedades Descrição Exemplo
Obrigatório Todas as propriedades são necessárias, a menos que especificado de outra forma. firstName: string;
Opcional Adicione um ponto de interrogação (?) ao final do nome da propriedade. Use esse tipo para propriedades que não são obrigatórias. Isso impedirá que o sistema de tipos gere um erro se a propriedade for omitida. firstName?: string;
Somente leitura Adicione a palavra-chave readonly na frente do nome da propriedade. Use esse tipo para propriedades que só devem ser modificadas quando um objeto é criado pela primeira vez. readonly firstName: string;

Declarar uma interface com membros

  1. Abra o Playground e remova qualquer código existente.
  2. Declare uma interface chamada IceCream que inclua duas propriedades: flavor como um string e scoops como um number.
    3. TypeScript interface IceCream { flavor: string; scoops: number; }
  3. Agora, você pode implementar a nova interface. Vamos começar usando a interface IceCream como um tipo em uma declaração de variável. Declare uma nova variável chamada myIceCream como o tipo IceCream e atribua valores às propriedades necessárias. Observe que, à medida que você começa a digitar os nomes de propriedade, o IntelliSense sugere os nomes e tipos corretos. Verifique seu trabalho exibindo o valor no console.
    4. TypeScript let myIceCream: IceCream = { flavor: 'vanilla', scoops: 2 } console.log(myIceCream.flavor);
  4. Selecione Executar. Observe que a variante é exibida na janela de Log.
  5. Em seguida, criaremos uma função na parte inferior chamada tooManyScoops que usa a interface IceCream como tipo de parâmetro. Essa função verifica o número de scoops no objeto IceCream e retorna uma mensagem com base no resultado. Para testar o seu trabalho, passe o objeto {flavor: 'vanilla', scoops: 5} como um parâmetro e verifique o resultado retornando-o para o console.
    6. TypeScript function tooManyScoops(dessert: IceCream) { if (dessert.scoops >= 4) { return dessert.scoops + ' is too many scoops!'; } else { return 'Your order will be ready soon!'; } } console.log(tooManyScoops({flavor: 'vanilla', scoops: 5}));
  6. Execute o código clicando em Executar. A saída será parecida com a exibida abaixo.
    7. Console "vanilla" "5 is too many scoops!"
  7. Abra o painel JavaScript e observe as diferenças entre o código JavaScript e o código TypeScript. A interface não tem nenhuma representação no código JavaScript.
  8. O que acontecerá se você adicionar outra propriedade à interface IceCream? Vamos adicionar uma nova propriedade chamada instructions e atribuí-la ao tipo string.
  9. Examine os erros que aparecem no seu código. Para resolvê-los, você deve adicionar a propriedade instructions aos detalhes de implementação da declaração de variável e adicioná-la como um parâmetro passado para a função. Por enquanto, torne a propriedade instructions opcional adicionando um ponto de interrogação ao nome da propriedade. Todos os erros devem ser resolvidos, porque a propriedade instructions não é mais exigida pela interface.
